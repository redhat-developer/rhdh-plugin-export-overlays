# RHIDP-14515: Rename homepage frontend plugin from dynamic-home-page to homepage

- **Jira:** https://redhat.atlassian.net/browse/RHIDP-14515
- **Type:** Story (3 SP)
- **Status:** Review
- **Assignee:** Christoph Jerolimov
- **Team:** RHDH Frontend Plugins & UI

## Background

The homepage frontend plugin is currently named `dynamic-home-page` — a legacy name from when it was a thin wrapper around the upstream Backstage `home` plugin with added dynamic mount points. With RHDH 2.x / NFS, the scalprum dependencies and mount point code are no longer needed. The backend counterpart is already named `homepage-backend`, so the frontend should be aligned to just `homepage`.

The plugin ID also changes from `home` to `homepage` so it can be installed alongside the upstream home plugin.

## Scope

This plan covers **only the overlay repo** (`rhdh-plugin-export-overlays`). The upstream source changes in `rhdh-plugins` are a prerequisite and must land first.

## Prerequisites (upstream — rhdh-plugins repo)

- [ ] Rename npm package from `@red-hat-developer-hub/backstage-plugin-dynamic-home-page` to `@red-hat-developer-hub/backstage-plugin-homepage`
- [ ] Change plugin ID from `home` to `homepage`
- [ ] Remove scalprum dependencies and all mount point related code
- [ ] Remove OFS-related test apps, unit tests, and Playwright tests if NFS equivalents exist
- [ ] Publish the renamed package

## Risk Notes

- The package rename may be treated as a "new plugin" by Konflux, requiring new build pipeline configuration rather than just updating an existing one.
- Other workspaces (scorecard) have cross-references to the homepage plugin in their E2E configs — these must be updated in the same PR or coordinated.
- Release branches (`release-x.y`) should NOT receive this rename — it's a `main`-only change for the next major release.
